Ajax 요청을 통한 데이터 처리

    url : 특정 api 주소를 활용해서 데이터를 가져오거나, 컨트롤러 api 주소를 가지고 와서 데이터를 사용하거나 전달

    method : GET, POST, PUT, DELETE 등 이의로 다수 존재
             GET = 서버에서 데이터를 조회하거나 조회된 데이터를 가져올 때 사용 (읽기 DB = SELECT 데이터 처리 용어 = READ)
             POST = 서버에 데이터를 전송하거나 새로운 데이터를 추가할 때 사용 (전송, 생성 DB = INSERT 데이터 처리 용어 = CREATE)
             PUT = 서버에 기존 데이터를 수정하거나 전체 데이터를 교체할 때 사용 (수정, 교체 DB = UPDATE 데이터 처리 용어 = UPDATE)
             DELETE = 데이터가 요청 URL에 포함되거나 요청 본문에 포함될 수 있고, 서버에서 특정 데이터를 삭제할 때 사용 (삭제 DB = DELETE 데이터 처리 용어 = DELETE)

    success : api 주소나 backend에서 데이터 불러오기를 성공했을 때( = 데이터가 존재 하지만 결과가 없을 수도 있음 null 등) 실행되는 콜백 함수

                       api 주소에서 json 형식으로 나열된 데이터의 키 이름을 가져와서 키 이름 안에 들어있는 값을 보여줌
                    } else { 가져와서 조회한 데이터가 DB에서 존재하지 않을 경우
